## What is CSS?

- Cascading Stylesheets
- Style of the page

---

## Syntax

```jsx
selector {
  property: value;
  property: value;
}
```

---

## Concepts
- Where do we write styles?
- How do we style elements?

---

## Where do we write styles?

1. Inline
2. Style tags
3. Stylesheets

---

## Inline

```jsx
  <p style="color: blue">Don't do this</p>
```

<p style="color: blue">Don't do this</p>

---

## Style tags
```jsx
<html>
  <head>
    <style>
      p: {
        color: blue
      }
    </style>
  </head>
```

---

## Stylesheets

```jsx
<html>
  <head>
    <link rel="stylesheet" href="styles.css">
  </head>
```

---

### How do we style elements? Selectors
- **Type selector:** elementname
- **Class selector:** .classname
- **ID selector:** #idname
- **Attribute selector:** [attr=value]

---

## Combining selectors
- div.classname
- div, span
- div span
- div > span
- div ~ span
- div + span

---

## Specificity
Least specific to most specific

1. Type selector elementname
2. Class selector .classname
3. ID selector #idname

# Git

---

## What is Git? Why do we use it?

- Distributed version control system 
- Tracks changes in project files over time
- Coordinates work between multiple developers on project

---

## Concepts
- Repositories
- Commits and Staging
- Branches

---

## Repositories

<Box width={1/2}>

- Project tracked by Git
- **Local:** Isolated repository stored on your own computer
- **Remote:** Stored on server (Github), share your project code with others

</Box>

---

## Staging and Commiting
- Commits are snapshots of your projects
- Staging adds files to be commited

--- 

## Branches
- Individual timeline of project commits
- Master branch contains stable code
- Add new features in separate branches and merge

---

## Download Git

https://git-scm.com/downloads

---

## Setup

```jsx
git config --global user.name "Your Name"
git config --global user.email “your@email.com”
```
---

## Making a commit

```jsx
git init
git add file.js
git commit -m "Commit message"
```

---

## What's going on

---

## Useful commands

```jsx
git status
git log
```

---

## Creating a branch

```jsx
git branch <name>
git checkout <name>
```

---

## Merging changes
- Merge changes from a different branch into current branch

```jsx
git merge <name>
```
